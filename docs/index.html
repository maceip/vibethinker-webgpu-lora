Emberglass runs a fine-tuned Qwen2.5-3B reasoning model entirely on your own GPU, from a single static page, in hand-written WebGPU compute shaders. It thinks for thousands of tokens and streams a verdict — and nothing ever leaves your device. You can swap the model's specialization live by hot-swapping a LoRA adapter: the base loads once and never moves. Code: github.com/maceip/emberglass.

Bring your own model. This page is the engine — it hosts no multi-GB weights. To run it: serve a Qwen2.5-3B checkpoint from the same origin as this page (clone the repo, drop the model in ./model, npx http-server), then set the model URL below and press Load. Needs a browser exposing WebGPU with the subgroups feature (Chrome 128+ / Canary). Drag in a PEFT/MLX LoRA adapter to hot-swap a specialization.
